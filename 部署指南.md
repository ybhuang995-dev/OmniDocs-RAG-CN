# OmniDocs-RAG-CN MCP 知识库 — 部署指南

> 将 OmniDocs-RAG-CN 接入 Claude Code，让 AI Agent 能搜索你的本地文档。
> 如果你还没运行 `python install.py`，建议先跑安装脚本，它会自动完成大部分配置。

---

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows / macOS / Linux |
| Python | 3.10+ |
| 内存 | 8GB+（bge-m3 约 1.1GB，reranker 约 1.1GB） |
| 磁盘 | ~3GB 剩余空间（模型下载 + ChromaDB 索引） |

## 2. 一键安装

```bash
git clone https://github.com/ybhuang995-dev/personal-data-for-agents.git
cd personal-data-for-agents
python install.py
```

`install.py` 会自动完成：
1. 检查 Python 版本
2. 安装 pip 依赖（chromadb、sentence-transformers、jieba 等）
3. 检测 GPU 并安装 PyTorch（CUDA/CPU 自动适配）
4. 预下载 AI 模型（bge-m3 + bge-reranker，共 ~2.3GB，仅首次）
   - 优先使用 HuggingFace，失败自动回退到 ModelScope（国内可访问）
5. 自动检测并配置 IDE 的 MCP 连接

**非交互部署**（CI/CD、脚本）：
```bash
python install.py --yes          # 跳过所有交互确认
python install.py --force --yes  # 强制重装 + 非交互
```

核心依赖说明：
- `fastmcp` — MCP 协议实现
- `chromadb` — 向量数据库（本地持久化）
- `sentence-transformers` — 嵌入模型加载
- `jieba` — 中文分词（BM25 关键词搜索）
- `readability-lxml` — Mozilla Readability 网页正文提取

> ⚠ **注意**：HuggingFace（huggingface.co）在国内可能需要代理才能访问。如遇模型下载失败，请先配置代理后重试 `python install.py`。模型也可手动下载放入 HuggingFace 缓存目录（`~/.cache/huggingface/hub/`）。

## 3. 手动配置 MCP（如果 install.py 未自动配置）

在 IDE 的 MCP 配置文件中添加（Claude Code 的配置文件位于 `~/.claude/mcp.json`）：

```json
{
  "mcpServers": {
    "omnidocs-rag-cn": {
      "command": "python",
      "args": [
        "<你的项目路径>/personal-data-for-agents/server.py"
      ],
      "env": {
        "RAG_DOCS_PATH": "<你要索引的文档目录>",
        "RAG_DB_PATH": "<你的项目路径>/personal-data-for-agents/chroma_db",
        "HF_ENDPOINT": "https://hf-mirror.com",
        "HF_HUB_OFFLINE": "0",
        "TRANSFORMERS_OFFLINE": "0"
      }
    }
  }
}
```

参考模板：[mcp_config_cn.json](mcp_config_cn.json)

**重启 IDE 后**，9 个 MCP 工具自动加载：
- `index_documents` — 索引本地文件夹
- `search_docs` — 混合搜索（向量 + BM25 + 重排序）
- `rag_status` — 查看索引状态
- `list_collections` — 列出所有集合
- `list_indexed_files` — 列出已索引文件
- `remove_source` — 删除指定文件
- `delete_collection` — 删除集合
- `reindex_collection` — 全量重建索引
- `index_url` — 爬取网页/仓库

## 5. 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `RAG_DOCS_PATH` | server.py 父目录 | 扫描的文档目录 |
| `RAG_DB_PATH` | `<项目>/chroma_db` | ChromaDB 持久化路径 |
| `RAG_EMBED_MODEL` | `BAAI/bge-m3` | 嵌入模型 |
| `RAG_RERANK_MODEL` | `BAAI/bge-reranker-v2-m3` | 重排序模型 |
| `RAG_DEVICE` | `auto` | `cpu` / `cuda` / `auto`（有 CUDA 健康检查） |
| `RAG_DASHBOARD` | 不启用 | 设为 `true` 开启 Web 管理界面（端口 6280） |
| `RAG_WATCH_PATH` | 不启用 | 设为目录路径，文件变动时自动重索引 |
| `HF_ENDPOINT` | 不设置 | HuggingFace 镜像站（国内推荐 `https://hf-mirror.com`） |
| `HF_HUB_OFFLINE` | `0` | 设为 `1` 强制离线模式（禁止所有网络请求） |
| `TRANSFORMERS_OFFLINE` | `0` | 设为 `1` 强制 Transformers 离线（配合 HF_HUB_OFFLINE） |

## 6. 常见问题

### Q: 首次启动很慢？
A: 正常。sentence-transformers 会自动下载 bge-m3（~1.1GB）和 bge-reranker-v2-m3（~1.1GB），只下载一次。

### Q: 启动卡在 "CUDA device health check"？
A: CUDA 检测有 15 秒超时。如果 GPU 不兼容，在 MCP 配置的 `env` 中加 `"RAG_DEVICE": "cpu"` 强制用 CPU。

### Q: 索引目录下有很多无关文件？
A: 只建一个子目录专门放文档，把 `RAG_DOCS_PATH` 指向它。

### Q: 如何清空重建？
A: 对 AI 说"用 reindex_collection 重建索引"，或命令行 `python manage.py reindex`。

### Q: 如何索引网页？
A: 对 AI 说"用 index_url 爬取 https://xxx"即可。支持网页、GitHub 仓库（`github://owner/repo`）、npm 包、PyPI 包、ZIP 文件。

### Q: MCP 服务器启动报错 "ModuleNotFoundError"？
A: 依赖未装全，运行 `pip install -r requirements.txt`。

### Q: 如何在国内网络环境下使用？
A: `install.py` 会自动尝试 HuggingFace → ModelScope 两个下载源。如果仍然失败，在 MCP 配置的 `env` 中设置 `"HF_ENDPOINT": "https://hf-mirror.com"`。

### Q: 模型已缓存，但启动时仍报网络错误？
A: 在 MCP 配置的 `env` 中同时设置：
```json
"HF_HUB_OFFLINE": "1",
"TRANSFORMERS_OFFLINE": "1"
```
这会让 `server.py` 完全跳过网络请求，只使用本地缓存的模型。

### Q: Windows 终端运行 install.py 出现乱码或 UnicodeEncodeError？
A: 使用 `PYTHONIOENCODING=utf-8 python install.py`，或升级到最新版 install.py（已内置 UTF-8 修复）。
